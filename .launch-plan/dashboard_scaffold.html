Executive summary

The validation loop produced two materially different plans converging on one safer launch. Opus v1 wrote a broad refactor plan; GPT-5.5 v2 wrote a defensive ship plan. Opus v3 adopted ~85% of GPT-5.5's scope cuts and added five operational items GPT-5.5 missed. The combined v5 plan is shorter, sharper, and safer than any individual version.

Score progression: v1 = 67 (per GPT-5.5 critique), v2 = 86 (self), v3 = 88 (self). v4/v6 audits pending.

Headline finding: the codebase is more mature than initially assumed — Ziina, pgvector RAG, Inngest, checkpoints, scripture corpus, dashboard tabs, 3-step onboarding, geo-currency are all already wired. The Monday launch is not a platform build; it is a careful validation of what exists plus six well-scoped additions.

Sharpest disagreements across versions

Where the two models took materially different positions. Each item shows both stances and the consolidated resolution.

1. Refactoring orchestrator.ts before launch → GPT-5.5 wins
Opus v1 Extract 5 modules from orchestrator.ts (1992 LOC). Pure extractions with behaviour-equivalence rule. Target ≤1500 LOC. Each extraction is a separate commit, reverted if tests fail.
GPT-5.5 v2 Explicitly forbidden. Pipeline contains working checkpoint, paid-fallback guard, Inngest dispatch, rate limit, lock, trace. Splitting in the final 24 hours creates regression risk that the test suite cannot fully catch.
Resolved in v3: Adopted GPT-5.5's position. Frozen until after launch. Refactor moved to weeks 2-3 with golden report snapshots.
2. Citation hygiene strategy → Opus push wins
Opus v3 Citation truthfulness is a P0 launch-blocker. Manual audit on 3 reports. If chapter metadata is missing on >20% of retrieved chunks, change the prompt before launch to "cite source title only, no verse number." Hallucinated verses on a paid report damage trust irreversibly.
GPT-5.5 v2 Downgrade citation display copy from "verse" to "source reference" before launch if metadata is unstable. Treated as display polish.
Resolved in v3: Promoted to P0 with hard gate. Pending v4 validation that GPT-5.5 accepts the elevation.
3. next-intl scaffolding → GPT-5.5 wins, with Opus refinement
Opus v1 Scaffold next-intl with /en and /hi routes Monday, ship Hindi v1.1 in 10 days.
GPT-5.5 v2 No next-intl route restructuring before Monday. Locale prefixes touch routing, middleware, canonical URLs, auth redirects, payment returns, sitemap. Blast radius too large for Sunday night.
Resolved in v3: No routes Monday. But preparing messages/en.json for future use is fine if nothing reads from it on Monday (preparation ≠ activation). Hindi waitlist component ships.
4. Placeholder testimonials → GPT-5.5 wins outright
Opus v1 Three placeholder testimonial cards while real ones are sourced. If real ones aren't ready by Mon 06:00, swap to a "1,200 charts decoded this month" credibility bar (only if true).
GPT-5.5 v2 No fake testimonials. No fake "charts decoded" count. No fake scarcity. If proof is needed, show a sample report excerpt and a visible source footnote. Period.
Resolved in v3: Adopted GPT-5.5's bright-line rule. No fake proof of any kind.
5. Load test model → Both partly wrong; Opus pushback wins on specifics
Opus v1 k6 at 50 VU, target /api/reports/start p99 ≤800ms, ephemeris /full-day-data p99 ≤300ms, no 5xx. Aim 200 RPS as P1.
GPT-5.5 v2 Don't run 200 RPS against report generation. Wrong metric. Run 20 concurrent /api/reports/start with bypass, 50 RPS on /api/health, validate Inngest queueing. SLO follows runbook (P95 ≤180s, success ≥95%).
Resolved in v3: Adopted GPT-5.5's burst-test model. Added explicit /api/geo, mobile-cold landing, and 5×Ziina-intent tests. Total: ~5 min of work, prevents real outage.
6. Currency support — present or absent? → GPT-5.5 factually correct
Opus v1 Currency switcher absent. Multi-currency moved out of scope for Monday.
GPT-5.5 v2 Currency IS already wired. middleware.ts:13 auto-detects country and forwards x-currency. src/lib/pricing.ts:17 serves USD/INR/AED price tables. Pricing.tsx:85 fetches /api/geo. What is absent is the manual user-facing picker — not currency support itself.
Resolved in v3: Adopted GPT-5.5's correction. Manual picker = P2. Auto-geo-currency is launch-ready as-is.
7. Number of approval gates → Opus push, pending v4 validation
Opus v3 Three approvals: (1) positioning copy Sat eve, (2) staging Sun 22:00, (3) content audit Mon 04:00. The content audit converts "we have RAG + Lahiri code" into "we shipped a correct astrology product." A 20-minute approval that costs nothing if all is well.
GPT-5.5 v2 Two approvals: positioning copy + final staging smoke. Everything else runs from objective gates.
Pending v4: If Codex accepts that domain correctness needs a human eye on at least one paid report, the third gate stays.

Gaps and open risks

Items surfaced across v1–v3 that remain open or partially addressed.

Open — content correctness

Hallucinated scripture verse on a paid report. Prompts ask for [[SOURCE:CHAPTER:VERSE]]; if retrieved chunks lack stable verse metadata, the model can invent scholarly-looking citations. Mitigation in v3 §2.4 + §4.1 but un-verified until Mon 04:00 audit.

Open — first paid user

Mahadasha calculation off on first paid user. If a Vimshottari period calculation has an edge case (e.g. Ayanamsa boundary, very recent birth, leap-year quirk), the first paid user will encounter it. Mitigation: content audit on 3 reports + support fast-refund. Severity: very high if it happens.

Open — observability blind spot

Sentry/PostHog CSP misconfig silently drops events. If CSP doesn't include the right hosts, the SDKs initialize but events never reach the backend, and there's no error to alert on. Mitigation: deploy CSP update before SDK init, verify staging events arrive within 60s.

Open — support coverage

Aarsh asleep Mon 02:00–08:00 IST. Launch traffic likely peaks 09:00–14:00 IST; before that, monitoring posture depends on a single person. If a P0 issue lands at 03:00, response time could exceed SLA. Mitigation: stretch support window to 22:00 Sun → 23:59 Mon, but at risk of decision fatigue. Honest trade-off.

Partial — cost ceiling

Anthropic spend cap is informational, not enforced. If 100 paid users hit on Mon, no automatic kill switch exists. Mitigation: dashboard alert at 80%, but enforcement requires manual response. Acceptable risk at expected launch scale.

Partial — DPDP compliance

India's DPDP Act 2023 cookie consent timing. One-line consent banner is in v3 P0 if geo=IN, but specific consent text needs a legal review and that won't land before Monday. Mitigation: ship a minimal, conservative banner Mon; iterate based on legal review week 1.

Pending v4/v6: Codex's red-team audit will likely surface 2-4 more gaps not yet on this list.

Pattern observations

  • GPT-5.5 is observably better at file-level accuracy. Every claim in v2 came with a path:line. v1 had 5+ stale facts; v3 corrected them all after exposure to v2.
  • GPT-5.5 is observably better at ruthless triage. v2's 8-item P0 was cleaner than v1's 14-item P0. v3 ended at 9.
  • Opus is observably better at operational depth. Cost ceilings, support runbooks, performance budgets, compliance considerations all came from Opus v3 — items v2 didn't propose despite GPT-5.5 having repo access.
  • Opus is observably better at domain-protective framing. Citation truth as a launch-blocker (not display polish) came from Opus.
  • Both models converge fast. v3 adopts ~85% of v2's positions. The remaining 15% is where each model adds genuinely unique value.
  • The cheaper, dumber moves came from Opus v1. Broad refactor, fake testimonials, k6-against-LLM. GPT-5.5 caught all three.
  • The most valuable additions came from Opus v3. Citation gate, content audit, support runbook, cost caps — none of which v2 surfaced despite the same task brief.
